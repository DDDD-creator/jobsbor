## 2. 数据格式规范

### 2.1 JobSeed 接口定义

```typescript
interface JobSeed {
  id: string;                    // 唯一标识，格式: {行业前缀}{序号}
  title: string;                 // 职位标题，中文，15-30字
  slug: string;                  // URL标识，英文小写+连字符
  company: string;               // 公司名称，中文
  companySlug: string;           // 公司URL标识
  companyLogo: string;           // 留空字符串，由系统填充
  description: string;           // 职位描述，150-400字
  requirements: string;          // 职位要求，使用 • 分点，5-10条
  salaryMin: number;             // 最低月薪（人民币）
  salaryMax: number;             // 最高月薪（人民币）
  location: string;              // 工作地点
  type: 'full-time' | 'part-time' | 'contract' | 'remote';
  industry: 'finance' | 'web3' | 'internet';
  category: 'engineer' | 'product' | 'design' | 'marketing' | 'finance' | 'operations' | 'research';
  tags: string[];                // 3-6个标签
  publishedAt: string;           // 发布日期，格式: YYYY-MM-DD
  applyUrl?: string;             // 可选，申请链接
}
```

### 2.2 字段取值规范

#### industry（行业）
- `finance` - 金融（银行、证券、基金、保险、金融科技）
- `web3` - Web3/区块链（DeFi、NFT、基础设施、DAO）
- `internet` - 互联网（电商、社交、内容、工具）

#### category（职位类别）
| 值 | 说明 | 典型岗位 |
|----|------|---------|
| `engineer` | 技术 | 开发、算法、架构、测试、运维 |
| `product` | 产品 | 产品经理、产品运营、代币经济学 |
| `design` | 设计 | UI/UX、视觉、交互、品牌 |
| `marketing` | 市场 | 市场、增长、品牌、商务 |
| `finance` | 金融 | 投行、行研、量化、风控、财富管理 |
| `operations` | 运营 | 社区、用户、内容、活动 |
| `research` | 研究 | 行业研究、策略研究、DeFi研究 |

#### type（工作类型）
- `full-time` - 全职（默认）
- `part-time` - 兼职
- `contract` - 合同制
- `remote` - 远程（Web3公司常用）

#### 薪资换算规则
- 国内岗位：使用原始人民币数值
- 国外远程岗位：按 `1 USD = 7.2 CNY` 换算为人民币
- 年薪转月薪：除以 12
- 范围取整：按 1000 取整（如 35,000 而非 35,200）

### 2.3 CompanySeed 接口定义

```typescript
interface CompanySeed {
  id: string;                    // 格式: c{序号}
  name: string;                  // 公司中文名或官方英文名
  slug: string;                  // URL标识
  logo: string;                  // 留空
  description: string;           // 公司介绍，100-200字
  website: string;               // 官网URL
  location: string;              // 总部/主要办公地
  industry: 'finance' | 'web3' | 'internet';
  foundedYear?: number;          // 成立年份
  size?: string;                 // 规模：如 "50-200", "10000+"
  tags?: string[];               // 3-5个标签
}
```

### 智能体 9: 数据清洗与去重智能体

**任务目标**：清洗所有爬取的数据，执行去重和质量检查

**去重规则**：

```
1. 相同 company + title + location = 重复
   → 保留最新 publishedAt 的一条
   
2. 相似度 > 80% 的 description = 疑似重复
   → 人工标记，保留数据更完整的一条
   
3. 相同 companySlug 但不同 company 名称 = 需统一
   → 使用最常见的公司名
   
4. 相同 slug 但不同 id = slug 冲突
   → 修改 slug 添加序号（如 senior-frontend-2）
```

**数据清洗规则**：

| 问题类型 | 处理规则 |
|---------|---------|
| 薪资为 0 或 null | 标注为 "面议" 或根据行业平均值填充 |
| location 为空 | 根据公司总部填充 |
| description < 50字 | 标记为低质量，尝试补充 |
| requirements < 3条 | 标记为需补充 |
| tags 为空 | 根据 title 自动生成 3 个标签 |
| publishedAt 为 future | 修正为当前日期 |

**ID 重新编号规则**：
- 金融岗位：f8, f9, f10...（从现有 f1-f7 继续）
- Web3岗位：w8, w9, w10...（从现有 w1-w7 继续）
- 互联网岗位：i7, i8, i9...（从现有 i1-i6 继续）
- 公司：c11, c12, c13...（从现有 c1-c10 继续）

## 7. 注意事项

### 7.1 爬取限制
- 遵守各平台的 robots.txt 规则
- 控制请求频率，避免被封禁（建议 1-2秒/请求）
- 如遇到验证码，记录并跳过

### 7.2 数据隐私
- 不要爬取个人联系方式（手机号、邮箱）
- 不要爬取公司敏感信息
- 仅收集公开的招聘信息

### 7.3 异常处理
- 页面解析失败时记录 URL 和错误信息
- 字段缺失时标记为 "N/A" 或根据规则填充
- 数据冲突时保留最完整的一条
