## 生成规范

### 内容要求
1. **字数**: 每篇文章 5000字以上（中文字符）
2. **原创性**: 深度原创文章，不得抄袭
3. **专业性**: 内容要有深度，体现行业洞察
4. **结构**: 清晰的章节结构，包含引言、正文、结语

### 文章格式
```json
{
  "id": "article-XXX",
  "title": "文章标题",
  "description": "文章描述（200字以内）",
  "content": "# Markdown格式内容",
  "source": "Jobsbor原创",
  "author": "Jobsbor编辑部",
  "pubDate": "2026-04-14T12:00:00.000Z",
  "image": "",
  "link": "",
  "sourceCategory": "original",
  "sourceLang": "zh",
  "keywords": ["关键词1", "关键词2"],
  "tags": ["标签1", "标签2"],
  "metaDescription": "SEO描述"
}
```

### 提交流程
1. 生成文章内容
2. 追加到 `src/data/blog-posts.json`
3. 更新 `kimi-tasks/TASK-001-blog-content-generation.md` 进度
4. git commit 并 push 到 main 分支
