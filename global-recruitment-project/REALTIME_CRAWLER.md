# 🕷️ 实时爬虫系统 v3.0

## 🚀 系统升级完成

从**每小时爬虫**升级为**实时爬虫系统**，支持前端实时获取+后端守护进程双模式。

---

## 🎯 两种实时模式

### 模式1: 前端实时直连 (已部署)
- **URL**: https://jobsbor.vercel.app/real-jobs
- **机制**: 页面直接连接 RemoteOK API
- **刷新频率**: 每5分钟自动刷新
- **特点**: 
  - ✅ 用户打开页面立即看到最新职位
  - ✅ 后台自动检测新职位并提示
  - ✅ 支持手动刷新
  - ✅ 显示实时在线状态

### 模式2: 后端守护进程 (可选)
- **脚本**: `scripts/crawler/realtime-daemon.js`
- **机制**: 后台持续监控，检测新职位自动构建
- **检查频率**: 每5分钟检查一次
- **特点**:
  - ✅ 检测新职位自动触发构建
  - ✅ 构建冷却保护（10分钟内不重复构建）
  - ✅ 去重机制，避免重复职位

## 🎮 使用方式

### 用户端 - 实时查看职位
1. 访问 https://jobsbor.vercel.app/real-jobs
2. 页面自动从 RemoteOK 获取最新职位
3. 每5分钟自动刷新
4. 点击"立即刷新"手动获取最新数据
5. 有新职位时会显示红色提示徽章

### 管理员 - 启动守护进程（可选）

```bash
# 进入项目目录
cd /root/.openclaw/workspace/recruitment-site

# 方式1: 使用控制脚本
./scripts/crawler/daemon-control.sh start
./scripts/crawler/daemon-control.sh status
./scripts/crawler/daemon-control.sh stop

# 方式2: 直接运行
cd scripts/crawler
node realtime-daemon.js

# 方式3: 使用pm2（推荐用于生产环境）
pm2 start realtime-daemon.js --name crawler-daemon
```

## 🎨 前端功能

### 实时状态显示
- 🟢 **实时在线** - 成功连接到职位源
- 🔴 **离线模式** - 连接失败，显示缓存数据

### 新职位检测
- 检测到新职位时显示红色 `+N 新职位` 徽章
- 点击徽章清除提示
- 新职位自动排在列表顶部

### 自动刷新机制
1. **页面加载时** - 立即获取
2. **每5分钟** - 自动刷新
3. **页面重新可见** - 立即刷新（切换回标签页）
4. **手动刷新** - 点击按钮

---

## 🛠️ 守护进程功能

### 状态追踪
- `.crawler-state.json` 保存已知的职位ID
- 防止重复抓取相同职位
- 自动清理旧数据（保留最近1000个）

### 智能构建
- 检测到新职位才触发构建
- 10分钟冷却期，防止频繁构建
- 自动构建并部署

## 🚨 注意事项

### CORS 限制
- RemoteOK API 允许跨域请求 ✅
- 其他数据源可能需要代理

### 静态导出限制
- 前端模式：用户浏览器直接访问API ✅
- 后端模式：需要服务器运行Node.js ✅

### 构建限制
- Vercel 免费版有构建次数限制
- 守护进程的冷却机制就是为了节省构建次数
